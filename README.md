# MLOps Batch Job - Task 0

This repository contains a minimal MLOps-style batch job in Python designed to process OHLCV data. It demonstrates reproducibility, observability, and deployment readiness by reading a configuration file, computing a rolling mean on closing prices, generating a binary trading signal, and outputting structured metrics alongside detailed execution logs.

## Features

* **Reproducibility:** Deterministic execution enforced via a YAML configuration and a fixed random seed.
* **Observability:** Comprehensive logging capturing data validation, processing steps, and error traces.
* **Resilience:** Strict input validation and an overarching error handler that guarantees a structured JSON error output if the pipeline fails.
* **Deployment Ready:** Fully containerized using Docker for a zero-configuration run.

## Prerequisites

* Python 3.9+ (For local execution)
* Docker

## Local Run Instructions

1. **Set up a virtual environment (Recommended):**
   ```bash
   python -m venv venv
   source venv/bin/activate  # On Windows use: venv\Scripts\activate
   ```

2. **Install dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Execute the batch job:** Run the pipeline using the following command (no hardcoded paths are used in the script):
   ```bash
   python run.py --input data.csv --config config.yaml --output metrics.json --log-file run.log
   ```

4. **Review Outputs:**
   * Check `metrics.json` for the structured evaluation metrics.
   * Check `run.log` for the detailed execution timeline and validation steps.

## Docker Build & Run Commands

The application is completely containerized. The Docker container includes the dataset and configuration file, and will print the final `metrics.json` directly to the standard output upon completion.

1. **Build the Docker image:**
   ```bash
   docker build -t mlops-task .
   ```

2. **Run the container:**
   ```bash
   docker run --rm mlops-task
   ```

## Example `metrics.json` Output

Below is an example of the structured output generated by a successful execution of the pipeline:

```json
{
  "version": "v1",
  "rows_processed": 10000,
  "metric": "signal_rate",
  "value": 0.499,
  "latency_ms": 47,
  "seed": 42,
```

> **Note:** In the event of a failure, the pipeline will output an error JSON payload containing the version, an `"error"` status, and the specific `error_message` before exiting with a non-zero code.
